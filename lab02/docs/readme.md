# Пояснювальна записка до ЛР2 (Проєктування мехатронної системи)

## 1. Мета роботи
Мета лабораторної роботи — виконати проєктування мехатронної системи на рівні структури та функцій: визначити склад підсистем, інформаційні та енергетичні зв’язки, сформувати логіку керування і описати принцип роботи системи у вигляді схем (Draw.io) та текстового обґрунтування.

## 2. Об’єкт проєктування
Об’єктом проєктування є мехатронна система, що поєднує механічну частину (виконавчий механізм/кінематичний вузол), електропривід, датчики, контролер керування та програмну логіку керування. Система розглядається як сукупність підсистем, що реалізують:
- отримання вимірювань (сенсори),
- формування керуючих впливів (контролер),
- перетворення енергії у рух/дію (привід і механіка),
- забезпечення безпеки та діагностики (захист/контроль стану),
- взаємодію з оператором (індикація/ввід команд).

## 3. Функціональні вимоги (узагальнено)
Система повинна:
1) виконувати керований вплив на виконавчий механізм відповідно до заданого режиму;
2) забезпечувати зворотний зв’язок за ключовими параметрами (положення/швидкість/струм/температура — залежно від конфігурації);
3) мати режими: ініціалізація, робота, аварія/захист, зупинка;
4) забезпечувати базову діагностику: контроль живлення, відмов сенсорів, перевантаження приводу;
5) бути структуровано описаною у вигляді схем: структурна схема МС, функціональна схема, схема інформаційних потоків/алгоритму.

## 4. Склад мехатронної системи (типова структура)
**Механічна підсистема:** рама/корпус, передача (редуктор/пас/шестерні), виконавчий орган, кріплення, демпфери/обмежувачі.

**Електромеханічна підсистема (привід):** електродвигун (DC/BLDC/кроковий/серво), драйвер/перетворювач, елементи захисту, силова проводка.

**Сенсорна підсистема:** датчик положення (енкодер/потенціометр), датчики швидкості (за потреби), струму/напруги, температури, кінцеві вимикачі, інші спеціалізовані датчики.

**Керуюча підсистема:** контролер (МК/плата керування), ПЗ/алгоритм керування, інтерфейси зв’язку (UART/I2C/SPI/CAN — залежно від реалізації).

**Підсистема живлення:** джерело живлення, DC-DC перетворювачі, запобіжники, фільтрація, захист від перенапруг/переполюсовки.

**Інтерфейс оператора:** кнопки/перемикачі, індикація, або віддалений інтерфейс (ПК/термінал/пульт).

## 5. Принцип роботи (логіка)
1) **Ініціалізація:** перевірка живлення, ініціалізація контролера, перевірка сенсорів, встановлення початкового стану.
2) **Самотест/готовність:** контроль наявності сигналів датчиків у допустимих межах, очікування команди запуску.
3) **Режим роботи:** зчитування вимірювань → формування помилки/цілі → обчислення керуючого сигналу (наприклад, PID) → подача сигналу на драйвер приводу → механічна дія.
4) **Діагностика в процесі:** контроль перевищення порогів (струм/температура/відхилення датчика/втрата зв’язку).
5) **Аварійний режим:** вимкнення/обмеження приводу, сигналізація помилки, фіксація стану до ручного скидання або відновлення.
6) **Зупинка:** плавне гальмування або відключення приводу, збереження статусу.

## 6. Схеми, що додаються
У репозиторії присутні:
- **структурна схема МС** (підсистеми та їх зв’язки);
- **функціональна схема** (ланцюг “сенсори → керування → привід → механіка”);
- **схема потоків** (інформаційні/енергетичні потоки, сигнали, зворотний зв’язок);
- **алгоритмічна схема** (стани та переходи: Init → Ready → Run → Fault/Stop).

Вихідники схем: `drawio/`  
Експорт для перегляду: `schemes_exports/`

## 7. Висновок
У межах ЛР2 сформовано структуру мехатронної системи, визначено підсистеми та ключові зв’язки між ними, описано логіку роботи в основних режимах і підготовлено комплект схем у Draw.io та їх експортні версії для перегляду.
