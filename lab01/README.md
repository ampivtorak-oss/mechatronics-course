# Аванпроект: двоступеневий підвіс для мультиспектрального сенсора БПЛА (Мехатронні системи)

**Курс:** Мехатронні системи  
**Викладач:** к.т.н., старший викладач, Іван Володимирович Жежера  
**Тема лабораторної роботи:** Аналіз становища та створення аванпроекту  
**Варіант ТЗ:** №4 — підвіс для сільськогосподарського коптера (аналіз стану рослин)

---

## 1. Короткий опис моделі
Проєктується **двоступеневий (Pitch/Roll) підвіс прямого приводу (Direct Drive BLDC)** для стабілізації мультиспектральної камери під час польоту. Ціль — компенсувати вібрації та кутові коливання коптера й утримувати оптичну вісь у напрямку **«в надир»** для отримання даних, придатних до агроаналітики (водний баланс, азот/хлорофіл, біомаса). Керування підвісом виконується платою стабілізації, що приймає команди з польотного контролера по **S.BUS** або через **UART**; камера керується/синхронізується через **Trigger/Sync**.

---

## 2. Аналіз потреб та актуальність
Для агромоніторингу критичними є повторюваність зйомки, стабільність кута нахилу сенсора, мінімізація змазу та геометричних викривлень. Інтегровані комерційні рішення зручні, але обмежують вибір сенсорів/інтерфейсів і часто не дозволяють інтеграцію спеціалізованого обладнання. Кастомний 2‑осьовий підвіс дозволяє:
- адаптувати механіку під конкретну камеру (маса, габарити, центр мас);
- забезпечити сумісність із обраним польотним контролером через **S.BUS/UART**;
- зменшити масу та габарити порівняно з 3‑осьовими промисловими системами для задач «надирної» зйомки;
- спростити технічне обслуговування (демпфери, кріплення, кабель‑менеджмент).

## 4. Очікувані технічні характеристики (цільові)
- Робоча висота зйомки: **60–100 м**.  
- Ступені стабілізації: **2 осі (Pitch/Roll)**.  
- Режими керування: команди кутів/режимів через **S.BUS**; сервіс/налаштування/телеметрія через **UART**.  
- Корисне навантаження: **камера до 350 г** (включно з кріпленням/кабелем; визначається CAD‑компоновкою).  
- Діапазон кутів (проектно): **Pitch ±45°**, **Roll ±45°** (визначається механікою та кабель‑обмеженнями).  
- Конструкція: демпферна платформа + жорсткі рами осей; фіксація кабелів для зменшення паразитних моментів; доступ до сенсора для сервісу.

## 8. Доцільність розробки моделі
Розробка є доцільною, оскільки забезпечує:
1) сумісність із конкретною мультиспектральною камерою та можливість інтеграції нестандартних сенсорів;  
2) оптимізацію маси/габаритів під конкретний коптер і політний профіль;  
3) керування від польотного контролера через **S.BUS/UART** без прив’язки до закритої екосистеми;  
4) ремонтопридатність і модульність (демпфери, кріплення, площадка сенсора);  
5) підвищення якості даних (стабільне положення сенсора для надійної аналітики).
